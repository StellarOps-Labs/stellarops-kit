# Issue Quality Checklist

- [ ] The problem statement is specific and scoped.
- [ ] Expected behavior describes the end state.
- [ ] Likely files or folders are named.
- [ ] Stellar or Soroban relevance is explained when applicable.
- [ ] Acceptance criteria are testable.
- [ ] Difficulty is marked.
- [ ] Labels include type, domain, and triage status.
- [ ] The issue can be completed without private maintainer context.
