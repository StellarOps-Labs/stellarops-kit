# Stellar/Soroban Tasks

Good Stellar and Soroban contributor tasks connect concrete code changes to ecosystem concepts without assuming contributors already know the whole stack.

## Wallet tasks

Examples include wallet connection error states, network mismatch messaging, and clearer signing flow feedback. These tasks should include accessible UI behavior and component tests.

## RPC tasks

Soroban RPC tasks often benefit from documentation examples, environment variable samples, and clear testnet/local network distinctions.

## Transaction status tasks

Transaction status work should cover pending, success, failure, and timeout states. Acceptance criteria should name the states and the UI or helper behavior expected for each.

## Documentation tasks

Documentation issues should name the missing setup path, command, environment variable, or concept. Avoid broad requests such as "improve docs."

## Testing tasks

Testing tasks can cover contract ID helpers, transaction status components, wallet state transitions, or mock adapters for contract reads.
