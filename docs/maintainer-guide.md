# Maintainer Guide

StellarOps Kit helps maintainers prepare repositories before asking contributors to work on them.

## Using the toolkit

Start with a backlog idea. Fill in the issue generator fields, then review the quality score. Improve missing fields until the issue is contributor-ready. Use the label planner to add difficulty, domain, and triage labels. Copy the generated Markdown into GitHub.

## Preparing for contribution programs

Before submitting to issue-based contribution programs, review:

- README setup and project context
- CONTRIBUTING workflow
- Code of conduct and license
- Tests and CI
- Issue templates
- Labels and beginner tasks
- Stellar/Soroban relevance in issues

The repository does not need to be perfect, but contributors should be able to install it, understand available work, verify their changes, and submit a focused pull request.

## Maintainer operating rhythm

Keep a small set of maintainer-approved issues ready. Review stale issues monthly. Convert vague ideas into scoped tasks before labeling them as beginner-friendly.
