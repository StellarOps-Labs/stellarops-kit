# Contributor Workflows

Clear issues help contributors choose tasks they can complete. A good issue explains the problem, expected behavior, files to inspect, labels, and acceptance criteria.

## How contributors benefit

Contributors spend less time guessing and more time building. Difficulty labels help them choose appropriate tasks. Stellar/Soroban relevance explains domain context before they enter the code.

## How PR checklists help review speed

Pull request checklists remind contributors to link issues, run tests, update docs, and provide screenshots or logs when behavior changes. This gives maintainers a consistent review baseline and reduces repeated review comments.
